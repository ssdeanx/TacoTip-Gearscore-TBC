# PRD

## Problem statement

- What user or maintainer problem are we solving?

## Goals

- What are the specific goals we want to achieve?

## Non-goals

- What is out of scope?

## User stories

- Make as many as necessary atleast 5-10 so you have wide range.

## Success criteria

- Success criteria should be specific, measurable, achievable, relevant, and time-bound (SMART).

## Constraints

- Limitations should be clearly defined.
